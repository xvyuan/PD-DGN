# Dataset Preparation

This folder contains the datasets and preprocessing instructions for
**PD-DGN (Pocket-Aware Descriptor-Guided Dual-Gated Graph Neural
Network)**.

PD-DGN requires the following information:

-   compound canonical SMILES
-   protein amino acid sequences
-   affinity matrices
-   train/test split files
-   ESM-2 residue-level embeddings
-   protein pocket annotations
-   residue contact maps

Due to the large storage requirements, this repository does not include
the generated ESM-2 embeddings and fpocket pocket files. Users should
generate them following the instructions below.

------------------------------------------------------------------------

## Dataset Structure

Each dataset should follow the structure:

    dataset_name/
    |
    |-- Y
    |-- ligands_can.txt
    |-- proteins.txt
    |
    |-- folds/
    |   |-- train_fold_setting1.txt
    |   |-- test_fold_setting1.txt
    |
    |-- esm2_features/
    |   |-- protein_id_1.npy
    |   |-- ...
    |
    |-- pconsc4/
    |   |-- protein_id_1.npy
    |   |-- ...
    |
    `-- pocket/
        |-- protein_id_1.json
        `-- ...

File descriptions:

-   `Y`: affinity interaction matrix.
-   `ligands_can.txt`: canonical SMILES of compounds.
-   `proteins.txt`: amino acid sequences.
-   `folds/`: training and testing split files.
-   `esm2_features/`: ESM-2 residue embeddings.
-   `pconsc4/`: residue contact maps.
-   `pocket/`: binding pocket residue annotations.

------------------------------------------------------------------------

## 1. ESM-2 Feature Generation

PD-DGN uses residue-level protein embeddings generated by **ESM-2 (650M
parameter model)**.

Model:

    esm2_t33_650M_UR50D

Install:

``` bash
pip install fair-esm
```

Example:

``` python
import esm
import torch
import numpy as np

model, alphabet = esm.pretrained.esm2_t33_650M_UR50D()

batch_converter = alphabet.get_batch_converter()
model.eval()

data = [("protein_id", "MKKFFDSRREQ...")]

_, _, tokens = batch_converter(data)

with torch.no_grad():
    results = model(tokens, repr_layers=[33])

representation = results["representations"][33]

embedding = representation[0, 1:-1]

np.save(
    "protein_id.npy",
    embedding.cpu().numpy()
)
```

Generated files should be stored in:

    esm2_features/

Embedding dimension:

    (sequence length, 128)

------------------------------------------------------------------------

## 2. Pocket Generation Using fpocket

PD-DGN uses pocket-aware protein representation.

Binding pockets are generated using **fpocket**.

Official repository:

https://github.com/Discngine/fpocket

Installation:

``` bash
git clone https://github.com/Discngine/fpocket.git

cd fpocket

make
```

Pocket detection:

``` bash
fpocket -f protein.pdb
```

Extract pocket residues and save them as:

    pocket/protein_id.json

Example:

``` json
{
    "pocket1": [12, 25, 36, 48],
    "pocket2": [101, 120, 135]
}
```

Residue numbering must match the sequence indexing in `proteins.txt`.

------------------------------------------------------------------------

## 3. Protein Contact Maps

PD-DGN also requires residue contact maps.

Store contact maps as:

    pconsc4/protein_id.npy

Each file represents:

    (sequence length, sequence length)

Residue connections are generated using:

    contact probability >= 0.5

## Notes

-   ESM-2 embeddings must correspond exactly to the protein sequences.
-   Pocket residue indices must match protein sequence positions.
-   Missing generated files may cause preprocessing failures.
-   Large generated files are excluded from this repository.

After completing dataset preparation:

``` bash
python run.py
```

to start training and evaluation.
